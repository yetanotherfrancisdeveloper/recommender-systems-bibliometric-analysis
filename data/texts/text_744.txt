Some visualization tools circumvent this complexity by providing higher-level abstractions, such as Vega-Lite [48] and Plotly [28]. One solution is to abstract popular visualization examples into general-purpose templates [39], and generate programming recommendations based on these pre-deî€›ned structures (e.g., [39,54]). However, for complex languages like D3 that mix imperative and declarative speciî€›cation [49], creating templates is not a straightforward process [31]. Furthermore, prior work focuses on creating and using templates within the boundaries of a speciî€›c system [21,39], and it is not yet clear how templates can be generalized to code written outside of these pre-deî€›ned boundaries. In this paper, we propose a data-driven and user-driven approach to the generation, recommendation, and integration of visualization code templates. With our approach, we identify and extract common template structures using the code of actual D3 users mined from the web. Then, we leverage our understanding of how D3 users apply these templates in the wild to develop automated features to (1) determine which code snippets to recommend to a user as they write code in real time, and (2) integrate these code snippets automatically into live user code. As a result, we can avoid abstracting away the complexities of D3 while still providing transparent and customizeable recommendations to implement a userâ€™s desired functionality, even for user code that does not follow a pre-deî€›ned visualization template. We designed our approach to be general purpose and applicable to most visualization speciî€›cation languages, as long as the following properties hold true: there is a suî€œcient number of visualization examples from which meaningful code templates could be extracted; and the language can be represented as an Abstract Syntax Tree to support our program analysis techniques. As an example, both Vega [50] and Vega-Lite [48] meet these requirements.
